# Introduction

The Movie Landing Page is a pre-built landing page template designed to display information about movies. The landing page is built using Angular for the frontend and NestJS for the backend.

# Extends

This repository has been created using the **[Nestjs Starter Repo](/starter-repos/nestjs-starter-repo)** as its foundation.

> **NOTE** Checkout **[Nestjs Starter Repo](/starter-repos/nestjs-starter-repo)** for more information.

## Installation

Clone the Movie Landing Page with frontend in Angular and backend in NestJS from the Github repository using the following command:

```bash
$ git clone https://github.com/CodieGlot/ssg-server.git
```

```bash
$ git clone https://github.com/jpeseWang/ssg-web.git
```

> **NOTE** Github code: **[Backend](https://github.com/CodieGlot/ssg-server)** and **[Frontend](https://github.com/jpeseWang/ssg-web)**.

## Usage

The Movie Landing Page with frontend in Angular and backend in NestJS is a web application that displays information about movies. The frontend of the Movie Landing Page is built using Angular and comes with pre-built components and styles to help you get started quickly. The backend of the Movie Landing Page is built using NestJS and comes with pre-built APIs for retrieving information about movies from a Postgres database.

Here are some of the key features of the Movie Landing Page:

- **Responsive Design** - The Movie Landing Page is designed to work on all screen sizes, from mobile devices to desktop computers.
- **Pre-built Components** - The landing page comes with pre-built components such as a header, navigation menu, and footer.
- **Dynamic Content** - The landing page dynamically displays information about movies retrieved from the backend API.
- **User Authentication** - The Movie Landing Page comes with pre-built APIs for handling user authentication using JWT tokens.

To customize the content of the landing page, simply edit the source code files in the frontend directory. The app.component.ts file is the main component that renders the landing page, and you can add or modify components as needed.

> **NOTE** Consider exploring **[Netlify](https://www.netlify.com/)** and **[Vercel](https://vercel.com/)** as an option for deploying your project.

## Deployment

To deploy the Movie Landing Page with frontend in Angular and backend in NestJS, you can use any **cloud hosting service** that supports Node.js applications and Postgres databases. You can build the frontend and backend for production using the following command:

```bash
$ npm run build
```

> **NOTE** My deployment: **[Backend](https://ssg-server-git-main-codieglot.vercel.app/)** and **[Frontend](https://ssg-web-lilac.vercel.app/)**.

This will create a production-ready version of the application in the dist directory. You can then upload the contents of the dist directory to your cloud hosting service.
