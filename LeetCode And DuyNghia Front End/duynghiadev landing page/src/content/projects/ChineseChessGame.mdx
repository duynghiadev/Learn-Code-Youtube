# Introduction

The Chinese Chess Game is a two-player game that is based on the traditional Chinese game of Xiangqi. This game is built using C# MonoGame for the frontend and backend.

## Installation

Clone the Chinese Chess Game repository from the Github repository using the following command:

```bash
$ git clone https://github.com/CodieGlot/ChineseChess.git
```

> **NOTE** **[Github](https://github.com/CodieGlot/ChineseChess)** code.

## Usage

The Chinese Chess Game built on C# MonoGame is a two-player game that is based on the traditional Chinese game of Xiangqi. The game has a simple and intuitive interface that allows players to move their pieces on the board using the mouse.

Here are some of the key features of the Chinese Chess Game:

- **Two-Player Game** - The Chinese Chess Game is a two-player game that is played on a virtual board.
- **Traditional Chinese Game** - The game is based on the traditional Chinese game of Xiangqi, which has been played for over a thousand years.
- **Simple Interface** - The game has a simple and intuitive interface that allows players to move their pieces on the board using the mouse.
- **Multiplatform** - The game can be played on multiple platforms, including Windows, macOS, and Linux.

To play the Chinese Chess Game, simply launch the game from Visual Studio or by double-clicking on the executable file in the bin/Debug directory. The game will launch in a new window.

To move a piece on the board, simply click on the piece and drag it to the desired location. The game will automatically check for valid moves and highlight the available squares.

To customize the game, you can modify the source code files in the src directory. The ChineseChess.cs file is the main component that renders the Chinese Chess Game, and you can add or modify components as needed.

> **NOTE** Make sure to have the following software installed on your machine Visual Studio, Monogame and Git.

## Deployment

To deploy the Chinese Chess Game, you can use any platform-specific installer creation tool, such as WiX Toolset or Inno Setup. You can build the game for production using the following command:

```bash

```

This will create a production-ready version of the game in the bin/Release directory. You can then use an installer creation tool to package the game for distribution.
